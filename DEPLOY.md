# Установка сайта на собственный сервер

Эта инструкция поможет развернуть проект (сайт + сервер + база данных) на своём хостинге, независимо от платформы.

Проект состоит из трёх частей:
1. **Фронтенд** — сам сайт (React). Собирается в обычные HTML/JS/CSS файлы.
2. **Сервер** — Python-функции (вход, чаты, лента, истории, загрузка файлов).
3. **База данных** — PostgreSQL.

---

## Что понадобится на сервере

- **Node.js** 18+ — для сборки сайта
- **Python** 3.11 — для сервера функций
- **PostgreSQL** 14+ — база данных
- **S3-хранилище** (AWS S3, Yandex Object Storage, MinIO) — для фото/видео
- Веб-сервер (Nginx) — отдавать сайт и проксировать на сервер функций (рекомендуется)

---

## Шаг 1. База данных

1. Создай пустую базу PostgreSQL.
2. Пропиши строку подключения в переменную окружения:
   ```bash
   export DATABASE_URL="postgresql://user:password@localhost:5432/mydb"
   ```
3. Создай все таблицы автоматически:
   ```bash
   pip install psycopg2-binary
   python server/migrate.py
   ```
   Скрипт создаст все нужные таблицы. Повторный запуск безопасен.

---

## Шаг 2. Сервер функций

1. Скопируй `.env.example` в `.env` и заполни значения (база, ключи S3, при необходимости — MAX).
2. Установи зависимости и запусти сервер:
   ```bash
   pip install -r server/requirements.txt
   # подгрузи переменные из .env (например через export или менеджер процессов)
   uvicorn server.app:app --host 0.0.0.0 --port 8000
   ```
3. Проверь, что работает — открой `http://<твой-сервер>:8000/` — должен вернуться список функций.

Каждая функция доступна по адресу `http://<твой-сервер>:8000/<имя-функции>`:
`max-auth`, `chats`, `social`, `stories`, `media-upload`.

> Для постоянной работы запускай через systemd, Docker или pm2 — чтобы сервер не выключался.

---

## Шаг 3. Сайт (фронтенд)

1. В корне проекта создай файл `.env` и укажи адрес своего сервера:
   ```
   VITE_API_BASE=https://api.твой-домен.ru
   ```
   (или `http://<твой-сервер>:8000`, если без отдельного домена)
2. Собери сайт:
   ```bash
   npm install
   npm run build
   ```
3. Готовые файлы появятся в папке `dist/`. Загрузи их на любой веб-хостинг или отдавай через Nginx.

---

## Шаг 4. Nginx (рекомендуется)

Пример конфигурации: сайт + проксирование запросов к серверу функций.

```nginx
server {
    listen 80;
    server_name твой-домен.ru;

    # Сайт
    root /var/www/site/dist;
    index index.html;
    location / {
        try_files $uri /index.html;   # SPA: все пути на index.html
    }

    # Сервер функций
    location /api/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

При такой схеме в `.env` фронтенда укажи: `VITE_API_BASE=https://твой-домен.ru/api`

> Не забудь подключить HTTPS (бесплатно через Let's Encrypt / certbot).

---

## Шаг 5. Файловое хранилище (S3)

Загрузка фото/видео работает через S3-совместимое хранилище.
- По умолчанию endpoint в `backend/media-upload/index.py` — `https://bucket.poehali.dev`.
- Если используешь своё хранилище (AWS S3, Yandex, MinIO) — поменяй в этом файле `endpoint_url` и адрес CDN на свои.
- Ключи задаются в `.env`: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`.

---

## Переменные окружения (кратко)

| Переменная | Обязательна | Назначение |
|---|---|---|
| `DATABASE_URL` | да | Подключение к PostgreSQL |
| `AWS_ACCESS_KEY_ID` | да* | Ключ S3 (*для загрузки файлов) |
| `AWS_SECRET_ACCESS_KEY` | да* | Секрет S3 |
| `MAX_BOT_TOKEN` | нет | Вход через мессенджер MAX |
| `MAX_BOT_USERNAME` | нет | Имя бота MAX |
| `VITE_API_BASE` | да | Адрес сервера для сайта (фронтенд) |

---

## Проверка после установки

1. Открой сайт — должна загрузиться страница входа.
2. Зарегистрируй тестовый аккаунт (логин + пароль).
3. Войди под ним. Если вход прошёл — сервер и база связаны верно.

Если вход выдаёт ошибку — проверь, что сервер функций запущен и `VITE_API_BASE` указывает на правильный адрес.
